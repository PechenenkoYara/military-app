### Сторінка управління контактами та фільтрами
Ця сторінка надає користувачеві інтерфейс для додавання, перегляду та фільтрації контактів за спеціалізацією. Вона поділена на дві основні частини — ліва колонка з формою додавання контактів і таблицею, та права колонка з фільтрами.

### Додавання Контактів:
У верхній частині лівої колонки розташована форма для введення нового контакту. Вона включає три поля:

Ім'я: звичайне текстове поле, яке дозволяє ввести ім’я контакта.

Спеціалізація: поле, куди вводиться категорія/галузь, до якої належить контакт.

Номер телефону: поле для введення контактного номеру.

Після заповнення форми натисканням кнопки "додати" користувач може надіслати дані на сервер через POST-запит. Дані перевіряються на коректність у відповідному Python-обробнику перед додаванням у таблицю.

### Відображення Контактів:
У нижній частині лівої колонки знаходиться таблиця, що відображає всі контакти, додані користувачем. У таблиці три стовпці:

Ім’я

Галузь (спеціалізація)

Номер телефону

Якщо список контактів порожній, виводиться повідомлення "У вас ще немає контактів."

### Фільтрація Контактів:
У правій колонці розташована форма фільтрації. Вона містить набір checkbox-фільтрів, кожен із яких відповідає певній категорії:

Автомобілі

Медицина та медичне обладнання

Одяг і амуніція

Сітки

3D-друк

Кошти

Продукти

Інше

Користувач може обрати кілька категорій, після чого натиснути кнопку "фільтрувати", щоб переглянути лише ті контакти, які відповідають обраним критеріям.

### Адаптивність:
Сторінка має адаптивну верстку і добре відображається як на комп'ютерах, так і на мобільних пристроях. Зміни у стилях підключаються через медіа-запити @media (max-width: 768px), що дозволяє змінювати структуру колонок, розміри кнопок і полів введення відповідно до розміру екрана.

### Стилізація та Візуальні Елементи:
Основна кольорова гама: зелений, світло-сірий, білий.

Елементи, як-от кнопки, таблиця та контейнер фільтрів, мають згладжені кути, відступи та тінь, що створює приємний, легкий інтерфейс.

Кнопка додавання та фільтрації має стиль .small-button з характерним зеленим кольором фону і округлими краями.

### Сторінка редагування профілю користувача
Ця сторінка дозволяє користувачу оновити особисту інформацію, змінити пароль, додати або змінити фото профілю, а також відредагувати розділ "Про себе". Інтерфейс реалізований у вигляді двох колонок — ліворуч знаходиться зображення профілю, праворуч — форма редагування.

### Редагування Даних Користувача
У правій частині сторінки знаходиться форма з наступними полями:

Прізвище — текстове поле для введення або оновлення прізвища користувача.

Ім’я — текстове поле для імені.

Вид діяльності — галузь або спеціалізація користувача.

Старий пароль — поле, яке потрібно заповнити для підтвердження особи перед зміною пароля.

Новий пароль — новий пароль для облікового запису.

Підтвердження пароля — повторне введення нового пароля для підтвердження.

Про себе — велике текстове поле, де користувач може залишити коротку біографію, опис діяльності або іншу інформацію.

Фото профілю — поле для завантаження нового зображення (файл-інпут).

Після внесення змін натисканням кнопки "Зберегти зміни" користувач надсилає дані через POST-запит. Дані проходять перевірку в обробнику Flask (перевірка пароля, валідація, обробка зображення тощо).

### Адаптивність
Інтерфейс підтримує адаптивну верстку завдяки використанню flexbox та медіа-запитів:

На великих екранах поля форми розміщуються у двох колонках.

На екранах до 768px верстка змінюється на вертикальну — зображення переміщується над формою, поля займають всю ширину екрану.

### Стилізація та Візуальні Елементи
Основний фон: світло-сірий (rgba(238, 238, 238, 1)).

Поля вводу: світлий фон, округлі кути, тонка сіра рамка.

Кнопка "Зберегти зміни" має клас .small-button:

Темно-зелений фон

Білий текст

Округлі краї

Плавний ховер і курсор pointer

Кнопка "Скасувати" має сірий фон і вигляд звичайного посилання, але з таким же стилем як основна кнопка.

### Збереження Введених Даних
У разі помилки введення (наприклад, неправильне підтвердження пароля), форма не очищується. Завдяки передачі значень назад через render_template, усі введені дані зберігаються у відповідних полях. Це покращує UX і запобігає втраті введеної інформації.

## html_registr
Ця сторінка є частиною веб-додатку і призначена для реєстрації нових користувачів. Вона дозволяє користувачам створити обліковий запис, заповнивши необхідні поля. Після успішної реєстрації користувач може увійти до свого облікового запису.

### Опис роботи сторінки
Початкова форма реєстрації: Користувач потрапляє на сторінку реєстрації, де йому пропонується форма для введення особистих даних. Поля форми включають:

Ім'я: Поле для введення імені користувача (обов'язкове).

Прізвище: Поле для введення прізвища користувача (обов'язкове).

Сфера діяльності: Поле для введення спеціалізації користувача (необов'язкове).

Email: Поле для введення електронної пошти (обов'язкове).

Пароль: Поле для введення пароля користувача, яке повинно відповідати вимогам (мінімум 7 символів, поєднання літер і цифр).

Підтвердження пароля: Поле для введення пароля ще раз для підтвердження, що він введений правильно.

Перевірка правильності введених даних: Після того як користувач заповнює всі обов'язкові поля та натискає кнопку "створити", система перевіряє правильність введених даних:

Якщо паролі не збігаються, користувач отримує повідомлення про помилку.

Якщо форма заповнена некоректно, на сторінці з'являться відповідні повідомлення про помилки.

Flash-повідомлення: Система використовує механізм flash-повідомлень для відображення статусу форми:

Успішна реєстрація: Якщо всі дані введено коректно, користувач побачить повідомлення про успішну реєстрацію і буде перенаправлений на сторінку входу.

Помилки в даних: Якщо введені дані не відповідають вимогам, на сторінці з'являться повідомлення з описом помилок (наприклад, "Паролі не співпадають").

Виведення введених значень після помилки: Якщо користувач не заповнив форму коректно і була показана помилка, усі введені значення зберігаються в полях форми. Це дозволяє користувачу виправити лише помилки без необхідності повторно вводити всю інформацію.

Кнопка "створити": Після заповнення форми користувач натискає кнопку "створити". Цей крок запускає процес перевірки даних на сервері, і, якщо все вірно, реєстрація завершується.

Перехід до входу: Якщо користувач вже має обліковий запис, на сторінці є посилання на сторінку входу, де можна ввести логін і пароль для доступу до особистого кабінету.

### Як працює цей шаблон на сервері
Рендеринг шаблону: Під час обробки POST-запиту сервер перевіряє введені дані:

Якщо дані правильні, вони зберігаються в базі даних (залежно від реалізації на сервері), і користувач отримує повідомлення про успішну реєстрацію.

Якщо є помилки (наприклад, паролі не співпадають), сервер генерує повідомлення про помилку та передає їх на фронтенд для відображення.

Flash-повідомлення: Використовується для передачі повідомлень між сервером та клієнтом. Вони відображаються на сторінці у вигляді повідомлень, які можуть мати різні категорії (наприклад, 'error' або 'success').

### Що робити, якщо виникли помилки
Перевірте правильність введення паролів: Переконайтеся, що пароль та підтвердження пароля збігаються.

Перевірте формат email: Email має бути вірним та відповідати стандартному формату (наприклад, user@example.com).

Перевірте мінімальні вимоги до пароля: Пароль повинен бути довжиною понад 7 символів і містити букви та цифри.

Якщо помилки продовжують виникати, можна звернутися до адміністратора веб-сайту для отримання додаткової допомоги.


## login

Ця сторінка призначена для входу користувачів до особистого кабінету на веб-сайті. Якщо у вас вже є обліковий запис, ви можете увійти за допомогою свого email та пароля. Якщо ви ще не зареєстровані, є можливість створити новий акаунт.

### Опис роботи сторінки
Форма входу: Користувач потрапляє на сторінку входу, де йому пропонується форма для введення своїх даних:

Email: Поле для введення електронної пошти (обов'язкове). Необхідно ввести email, який був зазначений під час реєстрації.

Пароль: Поле для введення пароля (обов'язкове). Це має бути той самий пароль, який ви використовували під час реєстрації.

Перевірка правильності введених даних: Після заповнення форми користувач натискає кнопку "Увійти". Сервер перевіряє правильність введених даних:

Якщо email та пароль співпадають з даними в базі, користувач буде успішно увійшов до свого особистого кабінету.

Якщо дані введені неправильно (наприклад, неправильний пароль чи невірний email), на сторінці буде показано повідомлення про помилку.

Flash-повідомлення: Система використовує механізм flash-повідомлень для відображення повідомлень після спроби входу:

Успішний вхід: Якщо користувач ввів правильний email та пароль, він буде перенаправлений до особистого кабінету.

Помилка входу: Якщо введені дані неправильні, на сторінці з'явиться повідомлення про помилку, наприклад: "Невірний email або пароль".

Посилання на реєстрацію: Якщо користувач ще не має облікового запису, на сторінці є посилання "Створити" під формою входу, яке веде на сторінку реєстрації, де можна створити новий акаунт.

Кнопка "Увійти": Після заповнення форми користувач натискає кнопку "Увійти". Якщо дані вірні, система перевіряє їх і надає доступ до облікового запису.

### Як працює цей шаблон на сервері
Рендеринг шаблону: Під час обробки POST-запиту сервер перевіряє введені дані:

Якщо email та пароль правильні, користувач отримує доступ до свого облікового запису.

Якщо є помилка (наприклад, невірний email або пароль), сервер генерує повідомлення про помилку та передає його на фронтенд для відображення.

Flash-повідомлення: Використовується для відображення результату спроби входу. Повідомлення можуть бути різного типу:

Error (помилка): Якщо введені неправильні дані.

Success (успіх): Якщо вхід вдався.

### Що робити, якщо виникли помилки
Перевірте правильність введеного email: Переконайтеся, що email введено правильно, без пробілів на початку чи в кінці.

Перевірте пароль: Переконайтеся, що пароль введено правильно. Якщо ви забули пароль, скористайтеся опцією відновлення пароля (якщо така є на сайті).

Якщо помилки продовжують виникати, можна звернутися до адміністратора веб-сайту для отримання допомоги.

## Перевірка всіх даних, які були введені під час реєстрації, входу та зміни даних профілю

Валідація всіх введених даних відбувається за допомогою різних функцій, кожна з яких має на меті перевірку коректності введених користувачем даних на основі певних правил. Ось детальніше пояснення процесу валідації для кожного поля:

## Валідація Email:
Перевіряється, чи містить email символ "@", і чи він зустрічається тільки один раз.

Локальна частина (до "@") не повинна починатися або закінчуватися крапкою.

Частина після "@" (домен) також не повинна починатися або закінчуватися крапкою.

Домен має бути правильно розділений на частини, що не повинні бути порожніми.

Якщо email не проходить жодну з цих перевірок, він вважається некоректним і функція повертає False.

## Валідація Номера Телефону:
Перевіряється, чи номер телефону починається з "+" або "0".

Перевіряється, чи довжина номеру знаходиться в межах від 10 до 15 символів.

Перевіряється, чи номер містить лише дозволені символи: "+", "-", "#", "(", ")", "*", цифри.

Перевіряється, чи немає послідовностей однакових роздільників, наприклад, два знаки "-" підряд.

Якщо хоча б одна з перевірок не проходить, функція повертає False.

## Валідація Імені та Прізвища:
Перевіряється довжина введеного імені чи прізвища — воно не повинно перевищувати 150 символів.

Перевіряється, чи перша буква кожного слова є великою.

Перевіряється, чи не використовуються неприпустимі символи. Допускаються лише літери та роздільники, такі як дефіс, апостроф чи пробіл.

Перевіряється, чи кожен фрагмент, розділений пробілами або іншими дозволеними роздільниками, має хоча б два символи.

Якщо ім’я або прізвище не відповідає жодному з цих критеріїв, функція повертає False.

## Форматування Імені та Прізвища:
Якщо ім’я або прізвище є валідним, але потребує виправлення (наприклад, потрібно зробити перші літери великими), використовується функція форматування. Вона перетворює перші літери кожного слова на великі, а решту — на малі, враховуючи дозволені роздільники.

## Валідація Пароля:
Перевіряється, чи довжина пароля не менша за 7 символів.

Перевіряється, чи не містить пароль пробілів.

Перевіряється, чи містить пароль хоча б одну цифру.

Перевіряється, чи містить пароль хоча б одну літеру.

Перевіряється, чи містить пароль тільки літери, цифри та дозволені спеціальні символи.

Якщо пароль не проходить хоча б одну з цих перевірок, функція повертає відповідне повідомлення про помилку.

Цей процес валідації є необхідним для забезпечення коректності та безпеки введених даних. Якщо хоча б один параметр не відповідає вимогам, користувач отримає повідомлення про помилку і буде попросений виправити введені дані.


## Визначення спеціалізації контаків і закріплення контакту за певною категорією

Валідація всіх введених даних відбувається за допомогою різних функцій, кожна з яких має на меті перевірку коректності введених користувачем даних на основі певних правил. Ось детальніше пояснення процесу валідації для кожного поля:

## Валідація введених даних за категоріями:
1. Валідація даних за категорією (автомобілі, медицина, одяг, тощо):

Дані користувача проходять фільтрацію за категоріями на основі ключових слів, які зчитуються з відповідних файлів для кожної категорії (наприклад, "Автомобілі", "Медицина", "Продукти").

Для кожної категорії є файл, який містить набір ключових слів, що відповідають конкретним темам.

Введений текст користувача розбивається на окремі слова та порівнюється з цими ключовими словами.

Якщо хоча б одне слово з введеного тексту співпадає з ключовим словом у категорії, то цей ввід відноситься до цієї категорії.

2. Визначення категорії за введеними даними:

Якщо у введеному тексті є співпадіння з одним із ключових слів для певної категорії, то користувач отримує відповідну категорію.

Якщо введення не співпадає з жодною категорією, функція повертає категорію "Інше".

Це дозволяє автоматично категоризувати введені дані користувачем на основі тематичних ключових слів.

Валідація за допомогою функції filter_input():
Введений текст користувача перетворюється на нижній регістр і розбивається на окремі слова.

Кожне слово перевіряється на наявність у списку ключових слів для кожної категорії.

Якщо хоча б одне слово з введеного тексту містить ключове слово з категорії, то ця категорія додається до результату.

Якщо жодного співпадіння не знайдено, повертається категорія "Інше".

Такий підхід дозволяє забезпечити коректну класифікацію навіть при наявності помилок або неточностей у введених даних.

Валідація за допомогою функції does_fit():
Функція перевіряє, чи є спільні елементи між двома списками. Це корисно для визначення наявності спільних категорій або для порівняння різних наборів даних, таких як ключові слова або введення користувача.

Якщо хоча б один елемент зі списку list1 зустрічається в списку list2, функція повертає True. В іншому випадку повертається False.

Це дозволяє переконатися, чи мають два списки спільні категорії або чи відповідають введені дані певним вимогам.

## Структура файлів:
Для кожної категорії існують окремі текстові файли, що містять ключові слова для відповідної категорії. Наприклад, для категорії "Автомобілі" використовується файл cars.txt, для "Медицини" — файл medicine.txt тощо.

Кожен файл містить набір ключових слів, що відповідають певній тематиці, і ці файли зчитуються та використовуються функцією read_files_and_categorize() для оновлення глобальної змінної CATEGORIES.

## Як працює програма:
Функція read_files_and_categorize() зчитує ключові слова з файлів і категоризує їх у словник CATEGORIES.

Користувач вводить текст, який передається функції filter_input(). Текст обробляється, і визначається категорія, до якої належать введені слова.

Якщо введення не підходить жодній категорії, система автоматично відносить його до категорії "Інше".

Функція does_fit() дозволяє перевірити наявність спільних елементів між різними списками, що може бути корисним для порівняння категорій або ключових слів.

Цей процес дозволяє ефективно категоризувати введені дані користувача та автоматично визначати, до якої категорії вони належать.
